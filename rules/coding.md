# Coding Instructions

## Generic Coding Style Guide (All languages):
1. Keep code clean and readable
2. Avoid unnecessary complexity
3. Use meaningful variable and function names
4. Write comments to explain complex logic
5. Ensure code is well-structured and modular
6. Follow project's contribution guidelines
7. Avoid hardcoding values; use constants or configuration
8. Follow SOLID principles where applicable
9. Follow Given-When-Then structure for tests
10. Run tests to verify execution (prefer single test over full suite)
11. Check test results and verify successful execution
12. Use fluent style whenever is possible
13. Imports must be added atomically with the code changes in a single Tool invocation
14. Prefer parametrized tests over single case tests
15. Extract complex logic or calculations to separate named methods


## Script/Tool Creation:
When user requests "smart" scripts or automation:
- Include health checks, status monitoring
- Add color-coded output for clarity
- Implement safety confirmations for destructive actions
- Provide comprehensive help/usage information
- Include both simple and advanced usage modes
- Prefer feature-rich over minimal implementations
